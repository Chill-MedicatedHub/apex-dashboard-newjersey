# Chill Medicated — New Jersey Sales Dashboard

A self-contained dashboard for Chill Medicated's New Jersey operations.
The scraper pulls data from the **LeafLink V2 API** (using your own
API token, no scraping or browser automation); the dashboard reads
the result and renders KPIs, charts, and tables.

```
apex-dashboard-newjersey/
├── scraper.py          ← pulls data from LeafLink
├── refresh_loop.py     ← optional: re-pulls every 15 min
├── dashboard.html      ← the dashboard UI
├── sales_data.json     ← generated by scraper (don't edit, don't commit)
├── .env                ← your API token (create from .env.example)
└── requirements.txt
```

---

## 1. First-time setup (5 minutes)

### A. Install Python dependencies

```
pip install -r requirements.txt
```

### B. Generate a LeafLink API token

1. Log in to LeafLink
2. Go to **Developer Options** (under account settings)
3. Create / generate an API token
4. Copy the token immediately — it can only be viewed once

The user generating the token needs both **"Can Access Developer Options"** and
**"Manage Orders Received"** permissions.

### C. Create your `.env` file

Copy the template:

```
# macOS / Linux
cp .env.example .env

# Windows
copy .env.example .env
```

Then open `.env` in a text editor and paste your token:

```
LEAFLINK_TOKEN="paste_your_token_here"
```

Save the file. **Do not share it or commit it to git.** (`.gitignore` already excludes it.)

## 2. Run it

```
# 1. Pull data
python scraper.py

# 2. Start a local web server
python -m http.server 8000

# 3. Open the dashboard in your browser
http://localhost:8000/dashboard.html
```

The scraper will print a summary like:

## 3. Keeping data fresh

### Background refresh loop

Open a second terminal and run:

```
python refresh_loop.py
```

This re-pulls every 15 minutes (configurable via `REFRESH_MINUTES` in `.env`).
The dashboard auto-reloads its data periodically and will pick up new orders
without you doing anything.

### Or schedule it (cron / Task Scheduler)

Same patterns as the MA dashboard — see that repo's README for examples.

---

## 4. When the token doesn't work

If you see:

```
ERROR: LeafLink rejected the request (status 401).
Your API token is missing, invalid, or expired.
```

Generate a new token in **LeafLink → Developer Options** and update `.env`.
Tokens generally don't expire automatically but can be revoked by an admin.

If you see status `403`, the token works but the user lacks permission. Confirm
the user has **Can Access Developer Options** and **Manage Orders Received**
under Manage Users.

## 5. Configuration

All settings live in `.env`. See `.env.example` for the full list, but the
ones you'll most likely touch:

| Variable              | Default                  | Purpose                                            |
| --------------------- | ------------------------ | -------------------------------------------------- |
| `LEAFLINK_TOKEN`      | *(required)*             | API token from Developer Options                   |
| `LEAFLINK_DAYS_BACK`  | `365`                    | How much order history to pull                     |
| `LEAFLINK_STATE`      | `NJ`                     | State to filter to (use `""` for all states)       |
| `LEAFLINK_DOMAIN`     | `app.leaflink.com`       | Use `www.sandbox.leaflink.com` for the test env    |
| `REFRESH_MINUTES`     | `15`                     | Refresh loop cadence                               |

---

## 6. Differences from the MA dashboard

This dashboard is functionally identical to the Massachusetts one — same KPIs,
same charts, same filters. Two things differ behind the scenes:

1. **Data source.** MA pulls from Apex Trading (web scraping via session cookie).
   This one uses LeafLink's official, supported API with an auth token. Much
   more reliable and ToS-compliant.
2. **Brand color.** NJ uses the Chill Medicated purple/magenta palette to match
   the New Jersey market's branding.

Both dashboards produce a `sales_data.json` in the same shape, so the HTML/JS
is shared between them.

## Security notes

- Your API token grants read access to your LeafLink data. Treat it like a password.
- `.gitignore` excludes `.env` and `sales_data.json` from git by default.
- The dashboard runs entirely on `localhost`. Nothing leaves your machine
  except direct calls to `app.leaflink.com` made by the scraper.
- Generate a *dedicated* LeafLink user for the scraper if you want to limit
  the blast radius of a leaked token.
